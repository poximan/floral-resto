/restobar

/restobar/android/in/auth/login/request
/restobar/android/in/history/request

/restobar/android/in/auth/login/request

Suscribir:

ALTERNATIVA SI HIVEMQ CLOUD NO PERMITE ACL POR DEVICE ID

Publicar Android:
/restobar/android/in/#

Suscribir Android:
/restobar/android/out/+/#

Esta alternativa funciona, pero es menos estricta desde seguridad y aislamiento.

CRITERIO DE DISENO

- El estado actual se publica por fragmentos chicos.
- Dashboard y colas viajan por topics separados.
- Historicos viajan por requestId y por tipo de dato.
- Si se pierde una publicacion, se pierde un fragmento acotado y no un snapshot completo.
- El cliente Android puede suscribirse con comodines de 1 nivel y n niveles.

NOTA SOBRE EL PUENTE INTERNO

- La reduccion adicional de polling no requiere nuevas reglas en HiveMQ Cloud.
- El disparo fino entre `domain` y `mqtt-publisher` se resuelve internamente con eventos via PostgreSQL LISTEN/NOTIFY.
- Para el broker MQTT, las ACL sugeridas arriba siguen siendo suficientes.
